Preparation

1.1 Installation

1.1.1 PHP

  • Windows:
    • Download and unzip the zip file for lastest version from: https://windows.php.net/download
    • Rename $PHP_PATH$/php.ini-development file to php.ini
    • Add $PHP_PATH$ to PATH Environment variable
  • MacOS:
    • Download and install for lastest version from: https://php-osx.liip.ch
      $ curl -s http://php-osx.liip.ch/install.sh | bash -s 7.3
    • Export PATH
      $ export PATH=/usr/local/php5/bin:$PATH
  • To confirm PHP is installed on your machine:
    $ php --version

1.1.2 Visual Studio Code

  • Download and install from: https://code.visualstudio.com/download
  • PHP Server extension:
    • Install PHP Server (brapifra) extension from:
      https://marketplace.visualstudio.com/items?itemName=brapifra.phpserver
    • Edit in settings.json:
      • "phpserver.phpPath": "$PHP_PATH$/php.exe" (for Windows)
      • "phpserver.phpPath": "/usr/local/php5/bin/php" (for MacOS)
      • "phpserver.port": 5050
    • test.php file: <?php phpinfo(INFO_MODULES); ?>
      » Right-click .php file » PHP Server: Reload Server

1.2 MVC project

1.2.1 Cấu trúc thư mục

Giải thích:

  • Thư mục projectname là thư mục chứa project.
  • Thư mục assets gồm các file hình ảnh, javascript, css, ...
  • Thư mục controlers chứa các file định nghĩa các lớp controller, mỗi controller có nhiều hàm action tương tác với model và pass data (nếu cần) qua view để hiển thị.
  • Thư mục models chứa các file định nghĩa các lớp model, các lớp DAO chịu trách nhiệm thao tác với CSDL.
  • Thư mục utils chứa các file tiện ích như PHPMailer, ...
  • Thư mục views chứa thư mục layouts chứa template hiển thị chung của trang view.
  • Trang index.php là trang chủ nhận mọi request từ browser.
  • Trang routes.php là trang điều hướng request tới controller và action tương ứng.

1.2.2 Trang chủ

  • Là trang nhận mọi yêu cầu truy vấn lên server, bởi vậy mọi đường dẫn truy cập đều phải có dạng:
    http://domain/index.php?controller=XXX\&action=YYY
  • index.php file

1.2.3 Router

  • Sẽ chịu trách nhiệm load controller và call action tương ứng với hai biến $controller và $action.
  • routes.php file

1.2.4 BaseController cho các controller con

  • Là lớp cha của các lớp controller con của hệ thống, mục đích là để tái sử dụng biến $folder và hàm render().
  • controllers/base_controller.php file

1.2.5 Template cho các trang view

  • views/layouts/template.php file

1.2.6 DefaultController

  • controllers/default_controller.php file

1.2.7 Các trang view
